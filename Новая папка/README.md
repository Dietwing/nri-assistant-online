# НРИ-Помощник

Версия: 1.1.3

Приложение для ведения настольно-ролевого боя. Работает как настольное Windows-приложение, как обычная HTML-страница в браузере и как web-версия с онлайн-комнатой боя.

## Возможности

- создание и редактирование листов персонажей;
- хранение библиотеки персонажей;
- сортировка листов персонажей и списков команд по алфавиту;
- импорт и экспорт библиотеки в JSON;
- экспорт текущего состояния боя в JSON;
- импорт состояния боя из JSON с сохранением карты и позиций участников;
- добавление персонажей в две команды;
- расчет инициативы по Ловкости при добавлении персонажей в команды;
- ручное изменение очередности хода, инициативы, хитов, максимальных хитов и КД на поле боя;
- отдельная вкладка карты боя, связанная с текущим составом команд;
- настройка размера карты до 175×131 клеток;
- приближение и отдаление карты;
- загрузка изображения фоном карты и наложение сетки поверх него;
- токены участников боя, их перемещение по карте и счетчик пройденных клеток за раунд;
- окно действия между токенами на карте;
- сворачиваемые логи боя во вкладке карты;
- онлайн-комната боя через Firebase Realtime Database;
- подключение игроков с телефона к карте боя;
- ограничение игрока своим выбранным токеном;
- автоматические и ручные броски атаки, урона и лечения;
- проверка формул эффектов перед сохранением персонажа;
- проверка броска атаки против КД цели;
- журнал боевых действий;
- отмена последнего действия;
- вкладка справки с кратким описанием механик.

## Основные обозначения

- Хиты — здоровье персонажа.
- КД — класс доспехов. Атака проходит, если бросок атаки равен КД цели или выше.
- Инициатива — d20 + модификатор Ловкости.
- Очередь — позиция персонажа в порядке ходов; заполняется по инициативе и может редактироваться вручную.
- БМ — бонус мастерства.
- d20 — двадцатигранная кость для проверки атаки.
- Формула урона — запись вида `1d8+3`, `2d6`, `1d4-1` или обычное число. Ограничение: не больше `20d100`.

## Проверка

Базовые тесты логики запускаются командой:

```bash
npm test
```

Готовые файлы находятся в папке `dist`.

## Онлайн-режим

Для бесплатной демонстрации можно использовать статический хостинг и Firebase Realtime Database.

1. Создать проект Firebase.
2. Включить Realtime Database.
3. Для учебной демонстрации можно временно разрешить чтение и запись в правилах базы:

```json
{
  "rules": {
    "rooms": {
      "$room": {
        ".read": true,
        ".write": true
      }
    }
  }
}
```

4. Разместить файлы приложения на GitHub Pages, Netlify или Vercel.
5. Гейммастер открывает вкладку `Онлайн`, вводит адрес базы и код комнаты, подключается и публикует бой.
6. Игрок открывает страницу с телефона, вводит тот же адрес базы и код комнаты, выбирает свой токен и двигает его на карте.

Открытые правила подходят только для защиты, тестов и небольшой демонстрации. Для публичного использования нужны авторизация и более строгие права доступа.
